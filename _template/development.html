Разработчикам

Стандарт хранения данных базы данных MyDiary 1 версии.

Данные хранятся в Indexed Database.

Indexed Database - это база данных в браузере позволяющие хранить сложные объекты. Эта база данных работает асинхронно, т.е. операции с ней не блокируют сайт.

Базовая структура тега записи:

  • name — строка обозначающие название тега (без #)

  • id — Число обозначающие id тега записи (не влияет на импорт, удаляется при экспорте)

Тегами на этом сайте являются слово (последовательность не пробельных символов, отделённая от других пробельным символ), которая начинается с # и имеет длину больше 1

Базовая структура записи:

  • $type — строка обозначающая UUID 1 типа записи

  • $tags — массив из строк, обозначающих названия тегов (без #), облегчения фильтрации по тегам

  • $date — строка обозначающая дату в формате YYYY-MM-DD, к которой относится эта запись

  • $time — строка обозначающая время в формате hh:mm, к которой относится эта запись

  • $changed — количество миллисекунд с эпохи Unix, обозначающие дату создания

  • $created — количество миллисекунд с эпохи Unix, обозначающие дату последнего изменения

  • $id — Число обозначающие id записи (не влияет на импорт, удаляется при экспорте)

Все имена этих полей начинаются с $, что бы избежать конфликта имён

Базовая структура типа записи:

  • uuid — строка обозначающая UUID 1 типа записи, один UUID — один тип записи, добавление типа с тем же UUID — обновление этого же типа

  • id — Число обозначающие id типа записи (не влияет на импорт, удаляется при экспорте)

  • title — строка обозначающие человеческое название типа на русском языке

  • description - строка обозначающие человеческое описание типа на русском языке

  • icon — строка обозначающая название значка в https://icons.getbootstrap.com/, который используется для отображения значка типа

  • template - строка обозначающая vue—шаблон, который используется для отображения предварительного просмотра заметки

  • form_template - строка обозначающая vue—шаблон, который используется для модального окна в котором происходит присмотр и изменение всей нужной информации о заметке,

  • fields — объект обозначающей настройки добавленных в тип записи полей

    • tags — массив строк, элементами которого являются названия полей по которым происходит фильтрация тегу

    • search — массив строк, элементами которого являются названия полей по которым происходит полнотекстовый поиск

    • require — массив строк, элементами которого являются названия обязательных полей

  • page - объект обозначающей настройки типа записи, его отсутствие означает, что у типа нет своей страницы

    • title — строка обозначающие человеческое название типа на русском языке, используется для отображения ссылок на страницу типа

    • template - строка обозначающая vue—шаблон, который используется для страницы типа записи

Структура файла экспорта / импорта

  • version - строка, содержащие версию формата (одно натуральное число)

  • records - массив, элементами которого являются записи

  • types - массив, элементами которого являются типы записей

  • tags - массив, элементы которого являются строки, содержащие названия тегов (без #)
